Overview

This platform uses the NASA C-MAPSS (Commercial Modular Aero-Propulsion System Simulation) dataset, a widely-used benchmark for predictive maintenance research. It contains simulated run-to-failure sensor readings from turbofan jet engines. Each engine starts in a healthy state with slight manufacturing variation, develops a fault during operation, and degrades over successive flight cycles until failure.

Subset Used: FD001

C-MAPSS has four subsets (FD001-FD004) with varying fault modes and operating conditions. This platform uses FD001: a single fault mode (HPC Degradation — High-Pressure Compressor wear) under a single operating condition (sea level, static).

Data Files

train_FD001.txt

100 engines run to complete failure. Each row is one flight cycle with 26 columns: engine_id, cycle number, 3 operational settings, and 21 sensor readings. Used to train both models.

test_FD001.txt

100 engines with recordings cut off at an unknown point before failure. Same 26 columns. The task is to predict how many cycles remain.

RUL_FD001.txt

Ground-truth Remaining Useful Life for each of the 100 test engines (values range from 7 to 145 cycles). Used for evaluation only.

Columns Per Row (26 total)

Identifiers (2)

engine_id — which engine (1-100)

cycle — flight cycle number

Operational Settings (3)

op1, op2, op3 — flight regime parameters (altitude, Mach number, throttle resolver angle). In FD001 these are near-constant since there is only one operating condition.

Sensor Readings (21)

s1-s21 — physical measurements from different engine modules. See sensor detail table below.

Sensor Detail (14 selected for modeling)

Sensors s1, s5, s6, s10, s16, s18, s19 are excluded — they show near-zero variance in FD001 and carry no useful degradation signal.

Feature Engineering

For each of the 14 selected sensors, the pipeline computes:

  • ▶ Lag features — values at t-1, t-3, t-5, t-10 cycles
  • ▶ Rolling mean & std — over 5-cycle and 10-cycle windows
  • ▶ Delta — cycle-to-cycle change
  • ▶ Trend slope — linear regression slope over a 10-cycle rolling window
  • ▶ Cycle norm — normalized cycle position (0 to 1) within each engine's lifetime

Total: ~127 features per row after engineering.

What the Models Predict

RUL Forecast LightGBM Regressor

Predicts the Remaining Useful Life (RUL) in flight cycles — how many more cycles the engine can operate before failure. Capped at 125 cycles (engines far from failure are all labeled 125 to reduce noise).

Failure Classifier LightGBM Binary

Predicts the probability of failure within the next 30 cycles. A binary classification where RUL ≤ 30 is labeled as "imminent failure". The output is a probability (0-100%).

Anomaly Detection Threshold-based

Compares the predicted RUL against learned thresholds to compute an anomaly score. Levels: low, medium, or high. High anomaly scores trigger alerts.
